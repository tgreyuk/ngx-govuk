Install packages from npm

You will need to install "govuk-frontend" and "ngx-govuk" packages from npm. 


  


  

    
Setup GOV.UK styles and fonts


    

      This libary does not come packaged with any additional styles or style
      references. You will need to be include GOV.UK styles from node_modules
      into your project. It is assumed your style language is set to "scss".
    


    

      Please note that several sass warnings are generated in the console. These
      warning are coming directly from GOV.UK styles and would be present in any
      Angular GOV.UK project.
    


    
Add required class names to markup


    

      You must add "govuk-template__body" and "govuk-frontend-supported" class
      names to the body tag in your "index.html" file.
    


    


    
Add GOV.UK class names to body


    

      Some components require a class name. GDS appends with this with
      Javascript so components without JS enabled are unstyled. Ofcourse this is
      not relevant in an Angular app.
    


    


    
Include GOV.UK styles in your app


    

      There are two options for including
      GOV.UK styles
      into your project.
    


    

      Option 1: Include all GOV.UK styles in your project.
    


    

      To include all GOV.UK styles, in your "(angular | project).json" file, add
      the compiled CSS to the styles array. This is the quickest setup, however
      please note you may need (or prefer) to customise the output.
    


    


    

      Option 2: Customise GOV.UK styles in your project.
    


    

      If you are only using a subset of GOV.UK components you may prefer to
      import specific styles. For example if you are only using the header and
      footer components you can import only the component styles you need in
      your "styles.scss" stylesheet (update the path to node_modules as per your
      structure). You will also need to customise the output if you are not
      licensed to use GDS Transport font.
    


    

      The advised implementation is to pre-build the GOV.UK styles to css and
      then import the compiled css.
    


    

  


  

    
Configure fonts


    

      Once your styles are setup you will need to determine if you should use
      the GDS Transport font or not. Further information can be found in the
      GOV.UK Typeface guide.
    


    
Using an alternative font


    

      If you are not using the GDS Transport font, you will need to override the
      "$govuk-font-family" sass variable by placing it before the scss imports
      in "styles.scss". GOV.UK recommendeds to use an alternative typeface like
      Helvetica or Arial. If you do not follow this step your browser will try
      and load the GDS Transport font resulting in 404 console errors.
    


    

  


  

    
Consuming components


    

      Components are exported under secondary entrypoints which enforces a
      deterministic API and true tree-shaking capabilities.
    


    
Importing components into your app


    

      All components are exported as standalone components and should be added
      to the imports array of your standalone component or NgModule.
    

    

      In the following code examples, we want to use an imaginary component
      called "NgxGovukExample" in a standalone component. The component selector
      is named "ngx-govuk-example".
    

    


    

      For a complete list of components, please visit the
      components page.
    


    

      Using inputs and outputs, properties and methods.
    


    

      Each component has a documented set of inputs and outputs that are bound
      to the compoennt in the normal Angular way.
    


    


    
Using properties and methods.


    

      Some components also expose properties and methods. These can be accessed
      by getting a handle to the component instance using template variables.
