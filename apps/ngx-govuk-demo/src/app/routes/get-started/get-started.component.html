Get started

1. Install packages

You will need to install "govuk-frontend" and "ngx-govuk" packages from npm. 







  
2. Include GOV.UK styles


  

    This libary does not come packaged with any additional styles or style
    references. You will need to be include GOV.UK styles from node_modules into
    your project.
  


  

    There are two options for including
    GOV.UK styles
    into your project.
  


  
2.1 Include all GOV.UK styles


  

    To include all GOV.UK styles, in your "(angular | project).json" file, add
    the following to the styles array:
  


  


  
2.2 Include specific styles


  

    If you are only using a subset of GOV.UK components you may prefer to import
    specific styles. For example if are only using the header and footer
    components you can import only the component styles you need in your
    "styles.scss" stylesheet (you may need to update the path to node_modules).
  


  






  
3. Components usage


  

    Components are exported under secondary entrypoints which enforces a
    deterministic API and true tree-shaking capabilities.
  


  
3.1 Importing components into your app


  

    All components are exported as standalone components and should be added to
    the imports array of your standalone component or NgModule.
  

  

    In the following code examples, we want to use an imaginary component called
    "NgxGovukExample" in a standalone component. The component selector is named
    "ngx-govuk-example".
  

  

  

    For a complete list of components, please visit the
    components page.
  


  

    3.2 Component inputs, output, properties and methods.
  


  

    Each component has a documented set of input and outputs that should be
    consumed as standard.
  


  


  

    If you want to work with the properties and methods, you can get the
    component instance by using template variables.
